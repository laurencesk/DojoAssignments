Clients and Services

Computers connected to the Webare called clientsand servers.

  • Clientsare the typical Web user's internet-connnected devices(for example, your computer connected toyour WI-FI, or your phone connected to your mobile network) and Web-accessing softwareavailable on those devices(usually a web broser like FIrefox or CHrome).
  • Serversare computers that store webpages, sitesor apps.WHen a client device wants to access a webpage, a copy of the webpage is downloaded from the server onto the client machine to be displayed in the user's web brownser.

So What Actualy Happens?

  1. The broser goes to the DNS server and finds the real addres of the serverthat the website lives on.
  2. The browser sends an HTTP requestmessage to the server asking it to sent a copy of the website to the client.
  3. If the server approves the client's request, the server sends the client a "200 OK" message,which means "Permission granted! Here you go!", and then starts sending the website's files to the broser as a series of small chunks
  4. The broser assembles the samll chunks into a complete website and displays it to you!
